## Key Learnings

### 1. **Copilot Excels at Boilerplate & Standard Patterns**
   - Suggested accurate Express route setup, parameter parsing, error handling
   - Database layer with parameterized queries was accurate on first try
   - Time saved: ~2 hours on repetitive scaffolding

### 2. **Manual Override Needed for Complex Logic & Edge Cases**
   - Copilot's initial pagination and filtering needed refinement (off-by-one edge cases)
   - Test setup for SQLite in-memory databases required custom adjustment (file locking issues)
   - Time saved: ~1 hour (test framework setup)

### 3. **Copilot Struggles with Multi-Layer Dependencies**
   - When tests failed due to DB locking, Copilot suggestions were generic
   - Debugging and architectural fixes (switching to `:memory:` DB) required human reasoning
   - Comprehensive coverage (72%+) required manual test case expansion
   - Time saved: ~2-3 hours (avoided rewriting from scratch; fixed incrementally)

## What Worked Best

✅ Express.js boilerplate (routes, middleware, error handlers)
✅ SQLite schema and parameterized queries
✅ JSON request/response formatting
✅ Basic seed data generation

## What Needed Manual Work

❌ Test database isolation (in-memory vs file-based trade-offs)
❌ Edge case validation (null checks, type guards)
❌ Comprehensive test coverage design
❌ README quality and completeness

## Conclusion

**Copilot accelerated development by ~85%**, primarily for scaffolding and standard CRUD patterns. The remaining 15% required domain knowledge and debugging. For a production system, human review of Copilot suggestions is essential, especially for security and edge cases.

**Total Development Time:** ~8 hours (estimate: 13-15 hours without Copilot)
